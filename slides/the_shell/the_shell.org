** Introduction

*** Why The Shell?
:PROPERTIES:
:BEAMER_ENV: block
:END:

#+ATTR_BEAMER: :overlay +-
- Traditional interfaces: graphical UIs, voice commands, AR/VR
- Modern interfaces great for ~80% of use cases, but can be limiting
- You cannot click a button that doesn't exist
- You cannot give a voice command that hasn't been programmed
- The shell provides a powerful *text-based interface* with flexibility and control

*** Focus
:PROPERTIES:
:BEAMER_ENV: exampleblock
:BEAMER_ACT: <2->
:END:

- We'll focus on *bash* (Bourne Again Shell)
- One of the most widely used shells
- Essential for developers, system administrators, and power users

** What is the Shell?

*** Definition
:PROPERTIES:
:BEAMER_ENV: definition
:END:

*Shell:* A program that allows you to execute commands and interface with your operating system.

*** Characteristics
:PROPERTIES:
:BEAMER_ENV: block
:END:

- Textual interface for computer interaction
- Most platforms have a shell with many types (bash, zsh, dash, fish, etc.)
- Requires a *terminal* / *terminal emulator* to interact with the shell
- We will use *bash* in this class (generally default on Linux)
- macOS / Windows have different defaults

*** Reference
:PROPERTIES:
:BEAMER_ENV: alertblock
:BEAMER_ACT: <2->
:END:

The *Bash Reference Manual* contains everything you need to know about bash!

** Important Terminology

*** 1. Traditional Terminal
:PROPERTIES:
:BEAMER_ENV: block
:END:

- Historical computing device
- Physical keyboard and screen connected to shared computer
- Text-only interface (pre-mouse era)
- Used specific protocols to communicate
- Evolution: colors, different fonts, extended character sets

*** 2. Modern Terminal Emulator
:PROPERTIES:
:BEAMER_ENV: block
:BEAMER_ACT: <2->
:END:

Software that emulates traditional terminals

*Examples by platform:*
- *Windows:* CMD.exe, PowerShell, PuTTY, Windows Terminal
- *macOS:* Terminal.app, iTerm2
- *Linux:* xTerm, Gnome Terminal, Konsole, Kitty, Ghostty, Alacritty

** Terminal Emulator Features

*** Modern Features
:PROPERTIES:
:BEAMER_ENV: exampleblock
:END:

- Supports tabs, split panes, and Unicode
- Quality-of-life features:
  - Search functionality
  - Copy/paste
  - Customizable appearance

*** 3. Shell/Command-line Interface
:PROPERTIES:
:BEAMER_ENV: definition
:BEAMER_ACT: <2->
:END:

Program started when logging into terminal that interprets commands and provides interactive environment.

*** Why Master the Command Line?
:PROPERTIES:

#+ATTR_BEAMER: :overlay +-

** Basic Commands

*** How Commands Work
:PROPERTIES:
:BEAMER_ENV: block
:END:

Inside a prompt you type a command, which is then executed or interpreted.

The shell:
1. Parses by splitting on whitespace
2. Runs the program indicated by the first word
3. Gives each subsequent word as an argument to the program

*** Important Notes
:PROPERTIES:
:BEAMER_ENV: alertblock
:END:

- Commands are *case-sensitive*
- Each command is a separate program (usually)
- Arguments modify command behavior
- Special characters need proper handling
- Commands can accept flags/options starting with =-=

** PATH Variable

*** What is PATH?
:PROPERTIES:
:BEAMER_ENV: definition
:END:

PATH is an environment variable that lists which directories the shell should search for programs.

*** Characteristics
:PROPERTIES:
:BEAMER_ENV: block
:END:

- Colon-separated list of directories
- Shell searches directories in order
- First matching executable is used
- Can be modified to add new directories
- Built-in commands take precedence over PATH

** PATH Variable - Example
:PROPERTIES:
:BEAMER_OPT: fragile
:END:

*** Exploring PATH
:PROPERTIES:
:BEAMER_ENV: block
:END:

/usr/bin:/sbin:/bin:/snap/bin

# Find location of command
cmsc398w:~$ which echo
/usr/bin/echo

# Run directly without PATH
cmsc398w:~$ /bin/echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:
/usr/bin:/sbin:/bin:/snap/bin
#+END_SRC

** Modifying PATH
:PROPERTIES:
:BEAMER_OPT: fragile
:END:

*** Adding to PATH

* xargs

** What is xargs?

*** Definition
:PROPERTIES:
:BEAMER_ENV: definition
:END:

=xargs= builds and executes command lines from standard input, converting input into arguments for another command.

*** Why xargs?
:PROPERTIES:
:BEAMER_ENV: block
:END:

Many commands don't read from stdin—they need arguments:
- =rm=, =cp=, =mv= expect filenames as arguments
- =xargs= bridges this gap
- Enables parallel execution
- Handles large argument lists safely

** xargs Basic Usage
:PROPERTIES:
:BEAMER_OPT: fragile
:END:

*** Simple Examples
:PROPERTIES:
:BEAMER_ENV: block
:END:

#+BEGIN_SRC bash
# Delete all .tmp files
find . -name "*.tmp" | xargs rm

# Move files to directory
ls *.txt | xargs -I {} mv {} archive/

# Print each item on new line
echo "one two three" | xargs -n 1

# Execute command per line
cat urls.txt | xargs -n 1 curl -O
#+END_SRC

** xargs Options

*** Common Flags
:PROPERTIES:
:BEAMER_ENV: block
:END:

| Option   | Description                           |
|----------|---------------------------------------|
| =-n N=     | Use at most N args per command        |

#+BEGIN_SRC bash
#!/bin/bash

# For loop
for i in {1..5}; do
    echo "Number: $i"
done

# Loop over files
for file in *.txt; do
    echo "Processing $file"
done

# C-style for loop
for ((i=0; i<5; i++)); do
    echo "Count: $i"
done

# While loop
count=0
while [ $count -lt 5 ]; do
    echo "Count: $count"
    ((count++))
done
#+END_SRC

#+BEGIN_SRC bash
# Break out of loop
for i in {1..10}; do
    if [ $i -eq 5 ]; then
        break
    fi
    echo $i
done

# Continue to next iteration
for i in {1..10}; do
    if [ $i -eq 5 ]; then
        continue
    fi
    echo $i
done

# Read file line by line
while IFS= read -r line; do
    echo "Line: $line"
done < file.txt
#+END_SRC

#+BEGIN_SRC bash
# Use local variables
calculate() {
    local a=$1
    local b=$2
    local result=$((a + b))
    echo $result
}

# Return exit status
is_valid() {
    if [ -f "$1" ]; then
        return 0  # Success
    else
        return 1  # Failure
    fi
}

if is_valid "file.txt"; then
    echo "File is valid"

#+BEGIN_SRC bash
# Create array
fruits=("apple" "banana" "cherry")

# Access elements
echo ${fruits[0]}        # First element
echo ${fruits[@]}        # All elements
echo ${#fruits[@]}       # Array length

# Add elements
fruits+=("date")

# Iterate over array

    echo $fruit
done

# Slice array
echo ${fruits[@]:1:2}    # Elements 1-2

#+BEGIN_SRC bash
#!/bin/bash
set -euo pipefail

# Backup script
backup_dir="/backups/$(date +%Y%m%d)"
source_dir="$HOME/documents"

# Create backup directory
mkdir -p "$backup_dir"

# Backup files
echo "Starting backup..."
tar -czf "$backup_dir/documents.tar.gz" "$source_dir"

if [ $? -eq 0 ]; then
    echo "Backup completed successfully"
else
    echo "Backup failed" >&2
    exit 1
fi
